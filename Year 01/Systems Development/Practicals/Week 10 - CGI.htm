HTML forms

Copy the example form presented in the lecture from here, put it somewhere in your public_html directory, chmod it and check that you can load it - note that you'll get an error if you hit 'Go!' at the moment as the nonsuch.cgi program does not exist in your directory

CGI program test.cgi is available here.  This just prints out the parameters passed to it from a form. Load, rename to test.cgi and chmod (executable - chmod 755) - you don't need to understand how the program works - we will cover that next week

Modify the htmltest.html form action to link to test.cgi

Try it out from the browser!  (Remember to load from the server - NOT locally.  The URL must start 'http://www.bit.uwe.ac.uk/~<yourusername>/<filename>') - [replace the <...> text with whatever your username and file name are]

Experiment with adding and altering the form tags to get different input fields.  In particular try radio buttons (remember they are grouped) and checkboxes (N.B. if you are modifying the html file and use the back button on the browser to move back from the results page to the form, you must reload the form (the text fields will blank)!)

Try using HTML tables to align form components (this is probably the easiest way to get a good looking layout)

test.cgi uses GET (by default) - alter the form method to POST and see what happens
